



* cd MAIS O NOME DO REPOSITÓRIO/ARQUIVO LOCAL, COM AS PALAVRAS SEPARADAS POR TRAÇO (-) -> FAZ O GIT BASH ENTRAR NESSE REPOSITÓRIO E, ASSIM, TRABALHAR DENTRO DELE;

* git clone -> COLA O LINK DO REPOSITÓRIO DO GITHIB PARA O GIT BASH, DENTRO DO REPOSITÓRIO LOCAL;

* git status -> MOSTRA  ATUALIZAÇÕES DO PROCESSAMENTO DO GIT BASH;

* git add . OU git add -a -> ADCIONA TODOS OS ARQUIVOS NOVOS OU ATUALIZADOS, IDENTIFICADOS, MAS AINDA NÃO RECONHECIDOS NO CONTROLE DE VERSÃO. E ASSIM FICAREM APTOS 
A SEREM EMPURRADOS/SUBIDOS P/ O GITHUB; 

* git commit -m -> ESCREVER UM COMENTÁRIO/INFORMAÇÃO, ENTRE ASPAS DUPLAS (" ") A RESPEITO DO ARQUIVO QUE ESTÁ SENDO
INCLUIDO OU ATUALIZADO NO GITHUB;

* git push origin main -> EMPURRA/SOBE OS ARQUIVOS E ATUALIZÇÕES NO GIT BASH DO REPOSITÓRIO LOCAL, SUA MÁQUINA, PARA O GITHUB. (ETAPA FINAL)
